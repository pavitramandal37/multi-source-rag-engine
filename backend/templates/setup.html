{{ app_title }}

Follow these steps to run the full stack locally.

1. Prerequisites
  • Docker Desktop — docs.docker.com/get-docker
  • Ollama — ollama.com/download
2. Clone & configure 
git clone <repo-url>
cd api-support-ai-chatbot
cp env.example .env
# Edit .env — the defaults work for local Ollama
3. Pull Ollama models

Run these on the host (not inside Docker):

ollama pull llama3.2
ollama pull nomic-embed-text

The LLM is used for chat answers, query rewriting, HyDE, and CRAG grading.
The embedding model converts text to vectors for search.
nomic-embed-text outputs 768-dim vectors, padded to 1536 by the service.

4. Start the stack 
docker compose up --build

This starts the Django web server and the PostgreSQL + pgvector database.
Migrations run automatically on first start.
Visit http://localhost:8000 when you see Starting development server.

5. Add your first source

Via the UI: go to the Sources page and use the URL, PDF, or Markdown tab.

Via curl (JSON ingest for bulk docs):

curl -s -X POST http://localhost:8000/api/docs/ingest/ \
  -H "Content-Type: application/json" \
  -d '{
    "source_name": "My Docs",
    "docs": [{"title": "Hello", "content": "This is a test document."}]
  }'
6. RAG pipeline toggles

Set these in .env and restart the container:

VariableDefaultEffect
RAG_USE_HYDEtrueEmbed a hypothetical answer instead of the raw query (better retrieval on content-heavy docs)
RAG_USE_CRAGtrueGrade retrieved chunks and discard irrelevant ones (prevents hallucination)
RAG_CRAG_THRESHOLD0.6Minimum relevance score to keep a chunk
URL_CRAWL_DEFAULT_DEPTH2How many levels deep the URL crawler follows links
7. Known limitations (v1)
  • No streaming — answers arrive as one block. SSE support planned for v2.
  • No embedding cache — re-ingesting the same URL re-embeds all chunks.
  • URL crawler — rate-limited (0.5 s/page), capped at 50 pages per source. Does not parse robots.txt. Use only on sites you own or have permission to crawl.
  • No auth on ingest/delete endpoints — intended for localhost use only.
